Created 2018-03-26| Updated 2020-04-15|Python

个人近期使用Python时的一些踩坑笔记，简单记录下

1、关于多任务处理

前面并不懂多线程多进程的区别，抓起来感觉需要就用，但却没有太大感觉。详细了解了下，知道自己还是太莽了，做份笔记。

理论上讲：计算密集型任务适合多进程，IO密集型任务适合多线程。

多进程

  • 通常来说多线程相对于多进程有优势，因为创建一个进程开销比较大，然而因为在 python 中有 GIL 这把大锁的存在，导致执行计算密集型任务时多线程实际只能是单线程。
  • 由于线程之间切换的开销导致多线程往往比实际的单线程还要慢，所以在 python 中计算密集型任务通常使用多进程，因为各个进程有各自独立的 GIL，互不干扰。

多线程

在 IO 密集型任务中，CPU 时常处于等待状态，操作系统需要频繁与外界环境进行交互，如读写文件，在网络间通信等。在这期间 GIL 会被释放，因而就可以使用真正的多线程。

2、如何暂停运行

  • 插入input()语句，简单实用
  • 插入time.sleep(10) 意味着运行到这暂停10秒
  • 插入os.system("pause") 会弹出cmd窗口，放后台运行时可以当作一种提示

3、字符判断

字符串判断

判断函数 用途
str.isalnum() 所有字符都是数字或者字母
str.isalpha() 所有字符都是字母
str.isdigit() 所有字符都是数字
str.islower() 所有字符都是小写
str.isupper() 所有字符都是大写
str.istitle() 所有单词都是首字母大写
isinstance(num,int) 判断数据是否是整数
isinstance(num,float) 判断数据是否是浮点数

使用案例，执行的结果是布尔类型。
