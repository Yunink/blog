为了熟练运用相关知识，在一些平台上合理刷题,这次是在CG-CTF，包括Web、MSIC 和 CRYPTO

CG-CTF

南邮的CTF试炼平台，有一些基础的东西，曾经备战CTF时刷过，不是那种靠脑洞的恶心的类型，这里记录一下解题过程，顺便总结一下相关的姿势。

Web

简单

  • 签到题
    查看源码，找到flag：nctf{flag_admiaanaaaaaaaaaaa}

  • 签到2
    修改maxlength大于字符长度，提交得到flag：nctf{follow_me_to_exploit}

  • 这题不是WEB
    在线查看图片源码view-source:http://chinalover.sinaapp.com/web2/2.gif,查找ctf，得到flag：nctf{photo_can_also_hid3_msg}

  • 层层递进
    这是脑洞？在源码区找到同名目录SO，联系题目在404页面view-source:http://chinalover.sinaapp.com/web3/404.html找到flag: nctf{this_is_a_fl4g}

  • AAencode
    这是密码题吧，在根据题的名称和链接http://homura.cc/CGfiles/aaencode.txt,aaencode解码得到flag：nctf{javascript_aaencode}

  • 单身二十年
    burp抓包挨个回放在相应包中得到flag：nctf{yougotit_script_now}

  • php decode

    直接尝试运行是会报错的，这是因为PHP的eval函数，在一句话中常用也是比较脸熟。可以将eval改为echo，或者在CLsI后面插入exho $ZzvSWE;，都可以得到flag

  • 文件包含
    构造payload：http://4.chinalover.sinaapp.com/web7/index.php?file=php://filter/convert.base64-encode/resource=index.php,base64解码得到flag

  • 单身一百年也没用
    和前面的单身二十年差不多，burp抓包挨个回放在响应头中得到flag：nctf{this_is_302_redirect}

  • COOKIE
    按题目提示，将Set-Cookie: Login=0的值改为1即可得到flag：nctf{cookie_is_different_from_session}

  • MYSQL
    按提示在http://chinalover.sinaapp.com/web11/robots.txt得到源码如下

    既是屏蔽了id为1024的数据，那么我们要的就是在这了，不过需要绕过，前面对id进行了整形转换，所以可以用浮点型绕过web11/sql.php?id=1024.87,得到flag：nctf{query_in_mysql}

  • /x00

    nctf必须是数字且包含#biubiubiu，如果没有意外，就陷入死循环了。不过，ereg函数存在NULL截断的问题，可以使用%00截断正则匹配。
