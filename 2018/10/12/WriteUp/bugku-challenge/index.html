BugKu-CTF WP
Created 2018-10-12| Updated 2019-12-24|WriteUp
Post View:|Comments:

在bugkuctf平台上的一些刷题与挑战记录，暂时只记录了18年做的一些简单Web题的解答过程

1、Web2

简单的读源码，得到flag。

2、计算器

简单的改源码参数

3、Web基础$-GET

连读源码的操作都省了。。。按提示在地址栏简单的get传参就行

4、Web基础$-POST

和第三题类似，但post传参并不能在地址栏中进行，这里我们可以利用hackbar插件进行辅助得到flag。当然，不怕麻烦也可以使用工具抓包或上脚本进行。

5、矛盾

还是get传参，不过这个题涉及到is_numeric($value)函数，其中的value只要是数字或数字字符串就返回true，否者就返回false，但在这里就是需要num为1，所以我们必须构造一个值为1但又不能是单纯的数字的组合提交得到flag。

6、Web3

关闭无尽的弹窗读源码，发现一串unicode编码，解码得到flag。

7、域名解析

题目就是简单的：听说把 flag.bugku.com 解析到123.206.87.240 就能拿到flag。。。所以这里有几种思路，一种是直接改包进行IP欺骗，另个就是更改本地的host文件。

8、你必须让它停下来

打开连接后网页不断在刷新，按照提示关闭JavaScript让它停止刷新，查看源代码，没有flag，莫慌，在源码页面刷新几次会有惊喜的。当然，你要是嫌烦，直接抓包就是。

9、本地包含

打开连接看到的代码提示，有我们想要的flag.php，还可以看到REQUEST这个变量，稳稳的本地文件包含攻击，没跑的。解读php代码直接在地址栏构造?hello=file(“flag.php”)，轻松愉快。

10、变量1

分析可以知道这是一个代码审计题，通过GET 传参的方式执行if语句，所以开始挨个尝试php的几个超级全局变量，发现GLOBALS是可行的。

11、web5

流程走一套，简单尝试读源码，发现一大页JSFuck代码，复制到console中运行解决。

12、头等舱

这题没得啥子讲的，抓包吧。flag就在HTTPhead中。。。

13、网站被黑

这题就比较有意思了，页面很美，比起被挂了博彩什么的那种强多了。按照提示，那就开始咯，先用御剑扫扫网页。有一点收获，尝试一下发现是个webshell登录页面，人工尝试了几个，都没中，于是上字典暴力破解，发现是hack，登录得到flag。

14、管理员系统

这题我刚了很久，因为我已开始牟劲的以为是个注入问题，一直在搭配尝试。。。过了很久才开始注意到下面的提示，才注意到“本地”，才想到这可能是个IP欺骗。读源的时候发现有一串base64的代码，解码为test123，一般就是password了，搭配用户名admin提交，依旧报错，意料之中。打开控制台记录网络活动，右键选择编辑重发，添加X-Forwarded-For:127.0.0.1伪装成本地用户，提交后可以在html中发现flag。

15、web4

简单尝试行不通，读源码发现有两串url编码，还有一个eval字符串运算返回函数，大概就有方向了，将两串以及eval函数中加上的一段分别解码合并在一起，就比较明显了，将if后的那串字符提交得到flag。

16、flag在index里

这一题包含了本地文件包含漏洞的利用和PHP伪协议。解题的思路来源于前段时间学PHP了解到php://filter协议时看到的一篇大佬的文章 。在这道题里边，我们可以通过这个协议将源码以base64的形式输出再转化得到flag。

17、输入密码查看flag

看题目要求是输入五位数，说多也不多，直接使用Burp爆破，得到flag。

18、点击一百万次

也不是很多，大概就换个鼠标嘛，点嘛。。。看源码可以知道这里只要我们给它一个大于一百万的clicks就行。通过post传参，获得flag，轻松愉快。

19、备份是个好习惯

一套小流程下来发现没搞头，才八十又不想搞得太复杂，使用御剑扫扫网页，其中一个后缀index.php.bak的点进去后偷偷给我下载了一个index.php.bak的文件，打开发现是一串PHP源码，分析只要我们给它两个有相同md5编码结果的不同的值就行，使用get传参得到flag。

20、成绩单

是个sql相关的题，简单尝试后可以知道它的数据库长度为4，于是使用联合查询id=-1’ union select 1,2,3,4#看到三个位置都有回显。使用id=-1’ union select 1,user(),version(),database()#看看它的用户名，版本，数据库名。当然，也可以使用国外一个大神makman提供的一个可视化回显，如下。可以看到有两个表fl4g和sc可惜打不开，但flag应该就在fl4g中，直接用id=-1’ union select 1,2,3,group_concat(column_name) from information_schema.columns where table_name=0x666c3467#得到一个字段skctf_flag，查询id=-1’ union select 1,2,3,skctf_flag from fl4g#得到flag。
